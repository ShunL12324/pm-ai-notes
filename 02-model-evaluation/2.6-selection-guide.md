# 2.6 模型选型指南

产品经理视角的模型选择方法论。

## 选型维度

### 1. 能力匹配
- 任务类型（对话、代码、推理、多模态）
- 语言支持（中文、英文、多语言）
- 上下文长度需求

### 2. 成本考量
- API 调用成本（输入/输出 token 价格）
- 延迟要求
- 并发需求

### 3. 部署方式
- API 调用 vs 私有部署
- 数据隐私合规要求
- 基础设施成本

### 4. 生态与支持
- SDK 和工具链成熟度
- 社区活跃度
- 技术支持响应

## 主流模型对比

| 模型 | 优势 | 适用场景 |
|------|------|----------|
| GPT-4 | 综合能力最强 | 复杂推理、创意任务 |
| Claude | 长文本、安全性 | 文档处理、合规场景 |
| Gemini | 多模态、性价比 | 图文理解 |
| 通义千问 | 中文、成本 | 国内业务 |
| DeepSeek | 代码、推理、成本 | 技术类应用 |

## 评测实践建议

### 建立评测集
1. 收集真实业务场景的典型 case
2. 包含边界情况和困难样本
3. 定期更新（避免过拟合）

### 评测流程
1. 先用公开 Benchmark 筛选候选
2. 用业务评测集精细对比
3. 小规模灰度验证
4. 持续监控线上效果

### 注意事项
- 不要只看单一指标
- 考虑 worst case 表现
- 评估模型更新的稳定性
