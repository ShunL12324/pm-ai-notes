# 5.3 MCP vs Skills 对比

两种 AI 能力扩展机制的深度对比分析。

## 架构理念差异

| 维度 | MCP | Skills |
|------|-----|--------|
| **核心理念** | 能力即服务 | 能力即知识 |
| **实现方式** | 独立服务进程 | 提示词注入 |
| **通信方式** | JSON-RPC 协议 | 无（直接注入） |
| **状态管理** | 服务端有状态 | 无状态 |

## 能力对比

| 能力 | MCP | Skills |
|------|-----|--------|
| **跨平台** | ✅ 支持多平台 | ❌ 仅 Claude Code |
| **实时数据** | ✅ 可访问外部服务 | ❌ 仅静态知识 |
| **工具调用** | ✅ 原生支持 | ✅ 通过 allowed-tools |
| **复杂逻辑** | ✅ 可编程 | ❌ 仅提示词 |

## 效率对比

| 指标 | MCP | Skills |
|------|-----|--------|
| **Token 消耗** | 较高（工具描述） | 较低（~28% 节省） |
| **延迟** | 有网络/进程开销 | 几乎无开销 |
| **启动成本** | 需要启动服务 | 无 |

## 开发体验

| 维度 | MCP | Skills |
|------|-----|--------|
| **开发语言** | TypeScript/Python | Markdown |
| **开发复杂度** | 中等 | 低 |
| **调试难度** | 需要日志/调试器 | 直接看文件 |
| **部署要求** | 需要运行环境 | 无 |

## 使用场景决策

### 选择 MCP 当...

1. **需要外部服务连接**
   - 数据库查询
   - API 调用
   - 云服务操作

2. **需要跨平台支持**
   - 同时用于 Cursor、Windsurf 等
   - 工具需要在不同 AI 产品中复用

3. **需要复杂逻辑**
   - 需要编程实现的功能
   - 有状态的操作

4. **需要实时数据**
   - 动态信息获取
   - 与外部系统同步

### 选择 Skills 当...

1. **工作流编排**
   - 组合现有工具
   - 定义操作流程

2. **指令封装**
   - 项目规范
   - 团队约定

3. **快速迭代**
   - 需要频繁调整
   - 不想维护服务

4. **Token 敏感**
   - 需要控制成本
   - 上下文窗口紧张

## 组合使用模式

**Skills 作为 MCP 的编排层**

```
Skills（工作流定义）
    ↓ 调用
MCP Tools（能力提供）
    ↓ 访问
外部服务
```

- Skills 定义"做什么"
- MCP 提供"怎么做"
- 两者互补而非替代

## 总结

| 选择 | 场景 |
|------|------|
| **MCP** | 需要连接外部世界、跨平台、复杂功能 |
| **Skills** | 工作流编排、团队规范、快速原型 |
| **两者结合** | 大型项目，Skills 编排 + MCP 能力 |
